The Cascade in CSS

Specificity Example

Specificity (Element VS. Class)

if the selector itself is very unique, then you should take into your cosideration that it's the applied. 

      
      .h1 {
        color: green;
      }

      h1 {
        color: yellow;
      }

Here, The Class Selector will dominate it
CSS Thinks always that you have a limited number of HTML elements that this class applied on it.

Specificity (Class VS. Attribute)

if the selector itself is very unique, then you should take into your cosideration that it's the applied. 

      
      h1 [class="example"] {
        color: green;
      }

      .h1 {
        color: yellow;
      }

Here, The Attribute Selector will dominate it
Attribute Selector selects a specific element by telling css what that attribute is and you can also determine the value that CSS should find!

Specificity (Attribute VS. Id)

if the selector itself is very unique, then you should take into your cosideration that it's the applied. 

      
      #h1 {
        color: green;
      }

      h1 [class="example"] {
        color: yellow;
      }

Here, The Id Selector will dominate it
You should using Id global attribute only with one single HTML element, so the id is the most specific element in CSS ever "in terms of Specificity not Cascade, There is something powerful on the way"!
